
В каталоге task_users находится SPA, которое необходимо доработать.
Для управления состоянием используется redux, данные в текущей версии получаем из JSON файла.
В каталоге test_api находится сервер. На разных этапах требуются разная версия api, для переключения нужно изменить параметр API_VERSION в файле test_api/.env и перезапустить сервер.

1) отрефакторить для использования апи вместо JSON-файла.
- используем запрос GET localhost:8888/users, данные храним в store.
- для авторизации используем POST localhost:8888/login { login: String, password: String }

2) приложение позволяет просмативать пользователей, переключать пользователей можно кнопками со стрелокой влево/вправо. Можно включать обор пользователей из определенной группы.
Этот функционал уже реализован, нужно проверить код на возможные ошибки, не оптимальные решения, при необходимости доработать, провести рефакторинг.

3) Необходимо реализовать возможность редактирования пользователей.
На сервер эти изменения не передаются, сохраняются только в сторе.
Редактирование должно быть доступно только авторизованному пользователю с ролью admin.
Достаточно проверки в компоненте на уровне if (role == admin) для дизейбла/скрытия кнопки редактирования

4) в меню нужно доделать переключение двух разделов - main и about

5) провести рефакторинг UI, например создать отделные компоненты для кнопок, текстовых полей...

6) перейти на работу с апи версии 2 с поддержкой сессий:
- в файле test_api/.env устанавливаем параметр API_VERSION=2 и перезапускаем сервер
- добавляем проверку сессии при использовании метода POST localhost:8888/login
- убираем параметр login из запроса localhost:8888/currentUser
- при использовании запросов, не забудьте добавлять в заголовки запроса credentials: 'same-origin'

7) Используя графики (D3, highcharts, charts.js), составить гистограмму пользователей и групп

8) использование WS
- в файле test_api/.env устанавливаем параметр API_VERSION=3 и перезапускаем сервер
- Подключится к WS ws://localhost:40510
- отправить запрос на получние новых пользователей (ws.send('getLiveDataUsers'))
- отображать новых пользователей в приложении

9) добавить unit тесты на jest для компонентов, экшенов, редьюсеров, селекторов
